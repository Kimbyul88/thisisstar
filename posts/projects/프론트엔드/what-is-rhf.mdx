## 1. 폼은 뭔가 어렵다



항상 프론트엔드 서비스 개발을 할때면 친해질 듯 안친해질것 같은 아이가 있다. 



바로 **Form** 이다. 



회원가입 할때 도 필요하고, 로그인도 하고, 

편집 툴을 만들 때도, 예약 서비스를 만들때도..

![image](/images/340598fa-b4d84b38767c.png)

사용자에게 정보를 받는 서비스를 주로 만드니까 필수적인 요소이다. 



내가 처음 프론트엔드 개발을 시작했을 쯤에는 Form을 어떻게 구현했었을까? 

내 기억에는… 



기억이 안나서 내 깃허브에서 긁어 왔다. 

```javascript
<form id="write-form" on:submit|preventDefault={handleSubmit}>
  <div>
    <label for="image">이미지</label>
    <input type="file" id="image" name="image" bind:files />
  </div>
  <div>
    <label for="name">이름</label>
    <input
      type="text"
      id="name"
      name="name"
      placeholder="이름"
      bind:value={name}
    />
  </div>
  <div class="submit-btn"><button type="submit">완료</button></div>
</form>
```



form 으로 label들을 전부 감싸고 input으로 하나하나 받는 형식.

`onChange`로 모든 요소 감지하고, `useState`로 저장하고, `onSubmit`으로 제출한다. 



 

난 이게 참 뭔가 싫었다. 



state로 object 관리해서 id 체크해서 저장하는 것도 맘에 안들었고, 

또 따로따로 state로 관리 하는 것도 귀찮았고



onSubmit 버튼도 preventDefault도 싫어서 걍 form 밖에 버튼 만들어서 submit 하곤 했다. 





HTML 다른 요소들은 다 친해지고 싶었는데 form 은 뭔가 친해지기 싫었다.

어떤 프로젝트를 시작하던지 Form을 만들어서 사용자들의 입력을 받아야 했다. 



왜 Form이 싫었을까? 생각해 봤다. 

<div class="callout">

1. 나의 통제 안에 있다고 생각이 안들었다. 계속 변경되다가 어느 순간 제출하면 끝?

그런데, React Hook 에는 useState의 재렌더링 특성과 정반대에 있는 훅이 있다.

## 3. 제어와 비제어 컴포넌트

useStae와 useRef는 제어와 비제어를 관장하는 대표적인 예시이다. 



### 제어 컴포넌트 (Controlled)

```javascript
const [input, setInput] = useState("");

const handleChange = (e) => {
   setInput(e.target.value);
};
  
<input value={value} onChange={handleChange}/>
```

- React state가 값을 관리한다.
- 직관적이다. 
- 하지만 리렌더링이 많다.


지금 까지 내가 하던 방식이 제어 컴포넌트 방식이다. 

사용자가 입력하기만 하면 바로바로 값이 실시간으로 함수에 전달되고,

바로바로 유효성 검사도 할 수 있다. 



하지만 !비효율 적이다! 



---

### 비제어 컴포넌트 (Uncontrolled)

```javascript
<input ref={inputRef}/>
```

- DOM이 값을 관리한다.
- 필요할 때만 읽는다.


즉, useRef를 사용하면 불필요한 렌더링을 줄이고 

입력값들을 필요할 때만 읽어서 한번에 제출할 수 있다. 



근데 React 공식문서에는 이런 문구가 있었다. 

<div class="callout">

<span style="background-color:#fbf3db">대부분 경우에 폼을 구현하는데</span> <span style="background-color:#fbf3db">[제어 컴포넌트](https://ko.legacy.reactjs.org/docs/forms.html#controlled-components)</span><span style="background-color:#fbf3db">를 사용하는 것이 좋습니다.</span> 제어 컴포넌트에서 폼 데이터는 React 컴포넌트에서 다루어집니다. 대안인 비제어 컴포넌트는 DOM 자체에서 폼 데이터가 다루어집니다.

</div>



이걸 `단일 진실 공급원(Single Source of Truth, SSOT)` 이라고 한다고 한다

state에 저장하면 실시간으로 언제든 값을 알 수 있지만, 

ref를 사용하면 DOM에서 꺼내와야한다. 



불필요한 리렌더를 하지 않는다는 성능적 장점이 있지만

값이 보존되지 않을 수 있다.

## 5. React Hook Form

근데 비제어 useRef를 그냥 사용하는 것은 좀 불편하다. 

ref를 직접 다루고, 관리하는것이 쉽지 않다. 



그래서 우리에겐 React Hook Form이 있다. 

![image](/images/340598fa-43aa1eb360bc.png)



React Hook Form은 비제어 컴포넌트 기반으로 동작한다. 

```javascript
<input {...register('email')} />
```



그런데 ! 제어 컴포넌트 처럼 사용할 수 도 있다. 

- 유효성 검사
- 오류 검사


제출도 한번에 한다. 

```javascript
handleSubmit((data) => console.log(data))
```



즉, 폼과 관련된 모든 값을 state에 들고 있을 필요가 없어지고

필요할때 필요한 것만 꺼내서 확인하면 된다.
