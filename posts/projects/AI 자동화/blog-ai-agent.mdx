## 1. 노션 DB 구축하기


노션에는 노션DB라는 것이 있다. 

바로 데이터베이스 표이다. 

일단 내 워크스페이스에 ‘블로그’를 생각해서 테이블을 구성한다. 


- projects
- classes
매우 쉽게 DB 구축 완료 ! 😌


여기서 중요한건 `status` 이다. 

이 자동화의 중요한 부분인 sync는 내 노션 DB에서 `status = published`인 포스트만 블로그에 넣는다. 

![image](/images/326598fa-2587c9bab9d7.png)

Writing, Draft, Published로 구분했다. 


## 2. 프로젝트 세팅

```xml
my-blog/
├── posts/                  ← MDX 파일들 저장 위치
├── scripts/
│   └── sync-notion.ts      ← 핵심 스크립트
├── .github/
│   └── workflows/
│       └── sync.yml        ← GitHub Actions
├── .env.local
└── CLAUDE.md               ← Claude Code용 명세서
```

여기서 제일 중요한건 `sync_notion.ts`이다. 


**`sync-notion.ts`** **가 하는 일:**

1. Notion DB에서 `Status = Published`인 글 목록 조회
1. 각 글의 블록 내용을 마크다운으로 변환
1. `posts/[...slug].mdx` 파일로 저장
1. GitHub에 커밋 + 푸시

`package.json` 수정하기

```json
{
  "scripts": {
    "sync": "tsx --env-file=.env.local scripts/sync-notion.ts"
  }
}
```

package.json에 이렇게 sync 스크립트를 추가해줘야 한다.

> 
> 
> ### 수정 내용 (scripts/sync-notion.ts)
> 
> 
> 1. Notion file 타입 이미지를 로컬로 다운로드해서 public/images/에 저장
> 
> 1. MDX에는 만료되지 않는 로컬 경로 (/images/파일명) 사용
> 
> 1. external 타입 이미지(직접 URL 입력한 것)는 기존대로 URL 그대로 유지
> 
> 1. .env.local 파일을 올바르게 로드하도록 dotenv 설정 수정
